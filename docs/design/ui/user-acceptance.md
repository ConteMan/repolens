# UI 用户验收记录（v1.2.0）

- 日期：2026-07-11
- 场景：首次使用者在本地 Git 工作树中使用 `repolens ui`。

## 已完成路径

1. 启动 `repolens ui --addr 127.0.0.1:8801`，打开页面。
2. 输入本地工作树绝对路径，成功读取 `.repolens.yml`。
3. 修改站点标题并新增一条 `docs/**` 规则。
4. 校验配置、查看 unified diff、确认写入。
5. 触发构建，页面显示构建完成与缓存输出路径。

该路径通过 Playwright 的真实浏览器自动化执行；写入和构建均由 UI API 完成，没有终端交互。

## 遇到的问题

- 自动化验收初次使用宽泛的“构建完成”文本断言，页面状态通知和构建阶段都包含该文案，导致测试选择器不唯一。改为精确定位构建阶段元素后通过。这不是用户操作阻断，但后续浏览器测试应避免宽泛文本定位。
- 端口 `127.0.0.1:8799` 已被先前本地进程占用时启动会失败；可显式传入其他 loopback 地址（例如 `--addr 127.0.0.1:8801`）继续使用。该行为符合 CLI 参数设计，后续可考虑改善占用端口时的提示文案。
