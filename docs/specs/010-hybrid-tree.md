# 010: 文件树混合形态（固定侧栏 ＋ 浮动覆盖层）

- 状态：已实现
- 关联：roadmap M5、spec 006、ADR-001、ADR-002
- 视觉契约：[docs/design/mockups/010-011-prototype.html](../design/mockups/010-011-prototype.html)（2026-07-08 三轮评审定稿，实现须与其对齐）

## 问题

v1 的文件树是与内容区耦合的固定侧栏：不能收起，占据阅读宽度；用户期望任何状态下都能唤出目录树导航，同时阅读时可获得全宽内容区。

## 行为

1. **宽屏（≥ 断点，默认 1024px）**：默认固定侧栏（现状）；顶栏最左侧树开关（☰）一键收起——侧栏**完全隐藏**（原型评审否决了保留细条 rail 的变体，2026-07-08 定稿），内容区占满；收起状态下点顶栏 ☰ 浮出**覆盖层树**（overlay，含遮罩，Esc/点遮罩关闭），覆盖层头部提供"固定"按钮还原为固定侧栏，点击树内链接跳转后覆盖层自动关闭。
2. **窄屏（< 断点）**：无固定侧栏，直接浮动形态。
3. **状态持久化**：收起/展开偏好存 localStorage（跨会话）；树的目录折叠状态与滚动位置沿用现有 sessionStorage 机制，固定/浮动两种形态共享同一份状态。
4. **覆盖范围**：全部 `view/` 页面（markdown / code / html 三模式 / image / binary / 目录页）行为一致。镜像层原样 HTML 不注入任何内容（ADR-001 不变）——全屏打开的原始 HTML 无树，属明确边界。
5. **无 JS 兜底**：退化为 v1 固定侧栏形态，可读可导航。
6. **树顶部预留搜索入口**（接线见 spec 012）。
7. **图标修正**（原型评审确认）：树的展开箭头弃用 `">"` 字符旋转，改为 CSS 绘制的 chevron（border 双边 45° 旋转，带 transition、respect reduced-motion）；Markdown 文件图标改为"带文本行的文档"形（与普通文件的折角形区分）。

## 接口契约

无 Go 接口变化。`PageData` 不新增字段；实现主体是 `internal/theme` 的模板（layout/tree）、site.css（浮动层、遮罩、断点）、site.js（开关逻辑与持久化）。site.js 体量预算从 ~200 行放宽到 ~350 行（010＋011 合计另计，见 011）。

## 边界与非目标

- 不做树的拖拽调宽（CSS 变量 `--sidebar-width` 已可定制）；
- 不做多级懒加载（全树仍构建期渲染进每页，仓库树过大的性能优化后置）；
- 镜像层永不注入。

## 验收

- 宽屏收起/展开/浮出三态切换正常，偏好跨页面、跨会话保持；
- 窄屏（模拟 < 断点）默认浮动，交互可用；
- 五种 Kind 页面 ＋ 目录页均可唤出树并跳转；
- 禁 JS 环境下退化为固定侧栏；
- 键盘可达：☰ 可 Tab 聚焦、Enter 触发、Esc 关闭覆盖层；
- 质量门禁 ＋ 相对链接自检通过；dogfood 视觉验收由维护者确认。
