# 014: UI 会话级构建输出目录

- 状态：已实现
- 关联：roadmap M8、specs 005 / 013、ADR-005（配置双信任域）、Issue #46

## 问题

`repolens ui` 当前只把工作树构建到用户缓存目录。该路径适合安全预览，但不能覆盖用户“构建到可直接检查或部署的本地目录”的任务；切换到 CLI 又会回到 Git `HEAD` 构建，无法直接复用 UI 中尚未提交的工作树状态。

本能力扩展 UI 的本地构建目标，但不改变 `output` 配置 schema、双信任域或 repolens 不参与部署的边界。

## 行为

1. **输出模式**：Build 页面提供“缓存目录（默认）”与“自定义目录”两种模式。缓存模式保持 Spec 013 的现有行为；自定义模式只接受用户输入的绝对本地目录。
2. **会话状态**：输出模式与自定义路径只存在于当前页面会话；打开另一个项目时恢复缓存模式。它们不进入配置表单、YAML diff、`.repolens.yml`、外部配置或浏览器持久化存储。
3. **工作树语义**：两种模式都继续通过 `source.Open(..., Worktree: true)` 构建，包含 UI 已确认写入但尚未提交的仓库配置与文件。
4. **路径规范化**：服务端要求绝对路径，解析现有路径组件中的符号链接并返回规范化后的最终路径。拒绝文件系统根、用户主目录、系统临时目录、用户缓存根、仓库根、仓库内部路径以及包含当前仓库的祖先目录。
5. **输出所有权**：不存在的目标和空目录可直接使用；带 `.repolens-build` 哨兵的已有目录视为 repolens 输出，覆盖前必须显式确认；已有文件、无哨兵的非空目录和无法安全检查的目标一律拒绝。
6. **原子发布**：构建先写入目标同级的临时目录，成功后再替换目标。替换前重新检查目标所有权，避免检查与发布之间的状态变化。构建或发布失败不得破坏目标中最近一次成功产物。
7. **交互**：自定义路径错误绑定到字段并恢复焦点；需要覆盖时显示完整规范化路径和明确确认对话框。构建结果区展示最终输出路径，并区分缓存输出与自定义输出。
8. **响应式**：1440px、1024px 与 390px 均可完成选择模式、输入路径、处理字段错误、确认覆盖和查看构建结果；路径和对话框不得造成页面级横向溢出。

## 接口契约

现有 `POST /api/build` 请求扩展为：

```json
{
  "path": "/absolute/repository",
  "output_path": "/absolute/output",
  "confirm_overwrite": true
}
```

- `output_path` 为空时使用 UI 管理的缓存目录；非空时启用自定义模式。
- `confirm_overwrite` 只授权覆盖当前规范化路径下、带 repolens 哨兵的已有输出；它不能绕过危险路径、仓库关系、文件或无所有权目录检查。
- 输出路径字段错误返回 `field: "output_path"`。需要确认时返回 HTTP 409、`code: "output_overwrite_confirmation_required"`；其他稳定错误码包括 `invalid_output_path`、`unsafe_output_path` 和 `output_not_owned`。
- 成功与 operation 查询响应增加 `output_mode: "cache" | "custom"`，`output_path` 始终为规范化后的最终绝对路径。
- 同一仓库同时最多一个 build operation，CSRF 与 loopback 限制保持不变。

## 与既有合同的关系

- 本 Spec 取代 Spec 013 中“用户自选部署输出目录”为非目标的部分；目录浏览器、内嵌 serve、预览 session 与部署集成仍不在范围内。
- 自定义输出是本地静态构建产物，不代表 repolens 参与发布、上传、托管或部署。
- `output` 仍是可信配置域；本功能是 API 请求级构建参数，不是仓库配置编辑能力。

## 非目标

- 修改 `output` schema、把输出路径写入仓库或跨进程记忆；
- 原生文件选择器、通用文件管理器、远程目录或网络存储；
- 自动上传、发布、托管或部署平台集成；
- 允许覆盖任意目录，或提供绕过哨兵与危险路径检查的开关。

## 验收

- 默认缓存构建行为与 Spec 013 兼容；自定义绝对目录可获得完整静态产物；
- 自定义路径不出现在 `.repolens.yml`、YAML diff 或配置 API payload；
- UI 未提交工作树内容参与两种模式的构建；
- 相对路径、危险根目录、仓库内外祖先关系、文件和无所有权非空目录均被拒绝；
- 已有 repolens 输出必须显式确认，且确认不能授权另一个路径；
- 目标在构建期间发生变化时，发布前复检并安全失败；
- 失败保留最近成功产物，同仓库并发构建仍返回 409；
- 桌面与 390px 浏览器覆盖字段错误、覆盖确认、成功、失败和无页面级横向溢出；
- `gofmt -l .`、`go vet ./...`、`go test ./...`、`go build ./...`、`pnpm check` 与 `pnpm test:e2e` 全绿。
