# ADR-001: 双层输出：镜像层 ＋ 浏览层

- 状态：已接受
- 日期：2026-07-03

## 背景

静态托管没有服务端逻辑，同一文件的不同视图（渲染页 / 原始内容）只能靠不同路径表达。同时需要：仓库内 HTML 文件的相对内链与资源引用可用；Agent 对每个文件有稳定纯文本 URL。

## 决策

输出分两层：**镜像层**（仓库文件原样复制、路径一字不改）＋ **浏览层**（`/view/` 前缀下生成的阅读页面）。

## 理由

- HTML 内链跳转靠"不破坏"实现：原路径原样托管，相对引用天然成立，无需解析改写用户 HTML；
- 镜像层天然是 Agent / curl 的 raw 访问层；
- 单层方案不成立：渲染后的 HTML 放在 `foo.md` 原路径，静态托管按扩展名返回 `text/markdown`，各平台浏览器行为不一致；
- 浏览层 URL 采用前缀式（`/view/<原路径>`）而非改写式（`/docs/foo.html`）：实现简单、与镜像层界限清晰、原路径可逆推。

## 后果

- 仓库内容在输出中存两份，对文档类仓库可接受；
- 浏览层内部指向 `.md` 的相对链接需在构建期改写为 `/view/` 路径；
- 全站必须只用相对链接，保证任意子路径部署成立。

## 修订记录

- 2026-07-04（细化，非推翻）：文件浏览页落地为 `view/<原路径>/index.html`，URL 为 `view/<原路径>/`（带尾斜杠）。原因：裸文件形式 `view/foo.md` 会与镜像层遭遇同一问题——静态托管按扩展名返回 `text/markdown`。本决策的核心理由（扩展名驱动 Content-Type 不可控）因此同样适用于浏览层自身。
