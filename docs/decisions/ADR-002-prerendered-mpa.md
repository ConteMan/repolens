# ADR-002: 浏览层采用预渲染 MPA ＋ 薄增强层

- 状态：已接受
- 日期：2026-07-04

## 背景

浏览层交互形态参照 docu.md（常驻文件树 ＋ 内容区即时切换 ＋ HTML 直出）。docu.md 是纯客户端阅读器（浏览器插件，本地实时渲染）；在静态站点上复现该体验有两条路：SPA 壳（shell 页面 ＋ 树 JSON ＋ 前端 fetch 镜像层并渲染）或预渲染 MPA。

## 决策

**预渲染 MPA ＋ 薄增强层**：构建期把每个文件渲染成独立页面，共享一份手写原生 JS 做文件树状态保持（sessionStorage）与可选 pjax 局部替换。

## 理由

- 项目核心是"定制渲染方式、规则、样式"，只有构建期管线有完整施展空间（链接改写、git 元数据注入、glob 规则），SPA 客户端渲染会废掉大半管线能力；
- 每页是真实 URL，无 JS 可读，任何静态托管零 hack（SPA 深链接在 GitHub Pages 等需 404 回退 hack）；
- docu.md 选客户端渲染是其产品形态（插件、本地即开）所迫，repolens 无此约束。

## 后果

- 浏览层输出体积随文件数线性增长，接受；
- 增强层 JS 必须保持"薄"：无框架、无打包器、无 Node 工具链；
- Mermaid 例外走客户端渲染（构建期渲染需 headless browser，不值得），vendor 本地 mermaid.js 按需懒加载。
