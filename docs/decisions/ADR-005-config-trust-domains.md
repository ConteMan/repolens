# ADR-005: 配置双信任域与级联规则语义

- 状态：已接受
- 日期：2026-07-04

## 背景

repolens 的价值主张是渲染**任意**仓库，包括不受构建者控制的他人仓库。仓库内配置文件因此是不可信输入。同时渲染定制需要一个"默认合理、局部覆盖"的规则模型。

## 决策

1. 配置分两个信任域：仓库内 `.repolens.yml` 只能影响渲染域（`site` / `ignore` / `render` / `rules` / `theme` / `view` / `agent`）；`source` / `output` / `access` 只认外部配置与 CLI，仓库内出现则忽略并告警。优先级：CLI > 外部配置 > 仓库内配置 > 默认值。
2. 规则语义为 editorconfig 式级联：按序全部匹配、后写覆盖先写，全局 `render` 段等价第 0 条规则。

## 理由

- 仓库内配置若能控制拉取来源或输出路径，渲染不可信仓库即产生任意写入风险；
- 外部配置必须存在，否则"仓库无需配合"的价值主张不成立（渲染他人仓库时无处放配置）；
- 级联优于 first-match：符合"默认全站合理、规则做局部修正"的心智模型，多条规则可正交叠加。

## 后果

- 配置加载器需实现两域合并与白名单校验；
- 密码等机密只允许引用环境变量名，配置文件可安全提交；
- 加密路径与 Agent 输出的交集需构建期 lint 告警（两者互斥）。
